Les Méthodes HTTP

( on parle aussi de verbes HTTP )

HyperText Transfer Protocol

Protocole de communication client-serveur pour le
World Wide Web

  Définition  

Les méthodes HTTP sont des actions
que l'on souhaite effectuer sur une ressource
via des requêtes

mais encore

Qu'est ce qu'une requête

Une demande effectuée par un client ( exemple de client : navigateur )

Qu'est ce qu'une ressource

Un fichier sur un serveur ( exemples de fichiers : index.html / contact.php / écétéra.écétéra )

Les méthodes HTTP sont donc l'ensemble des actions
qu'un client peut demander à exécuter ( via requête )
sur une ressource d'un serveur

Quelles sont ces méthodes/actions

Les méthodes

  • CONNECT

    ➡ Crée une communication bidirectionnelle avec la ressource

    CONNECT example.com:443

  • HEAD

    ➡ Accède aux métadonnées d’une ressource

    HEAD /index.html

  • OPTIONS

    ➡ Décrit les options de communication avec la ressource

    OPTIONS /index.html

  • TRACE

    ➡ Réalise un message de test aller/retour avec la ressource

    TRACE /index.html

Les méthodes

  • GET

    ➡ Demande une ressource

    GET /index.html

  • POST

    ➡ Envoie des données à la ressource

    POST /form.php

  • PUT

    ➡ Met à jour complètement une ressource

    PUT /update.html

  • PATCH

    ➡ Modification partielle d'une ressource ( ~ git diff )

    PATCH /partial.html

  • DELETE

    ➡ Supprime la ressource

    DELETE /old.html

✰    Précisions    ✰

Les méthodes peuvent être safe, idempotent, mise en cache ( cachabilité )

  • Safe : méthode qui ne modifiera pas la ressource
  • Idempotent : méthode qui retournera toujours le même résultat
  • Cachabilité : décharge le serveur, réduit traitement requête

Bonne pratique == Les méthodes safe sont à mettre en cache

Méthode HTTP Safe Idempotent Cachabilité
HEAD Oui Oui Oui
OPTIONS Oui Oui Oui
GET Oui Oui Oui
CONNECT Oui Non Non
TRACE Oui Non Non
PUT Non Oui Non
DELETE Non Oui Non
POST Non Non Non
PATCH Non Non Non
